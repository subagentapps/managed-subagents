## What works well (pattern to repeat)

- **Sequential dispatch with merge between** beats `dispatch all` for chains, because dirty-tree pre-flight checks fail mid-chain. The pattern `dispatch task X → mark-ready → merge → next` was reliable for 7 of 9 web subagents.
- **Brief plan upfront** + TDD-step granularity meant subagents shipped first try (no review iteration loops).
- **`--allow-protected` flag on merge** lets the orchestrator work past branch protection while still recording the merge correctly.
